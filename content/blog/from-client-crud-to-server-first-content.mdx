The original version of this site pushed too much responsibility into the browser:

- Content writes happened client-side.
- The rendering path fetched HTML blobs after page load.
- Security boundaries depended on Firebase configuration that was hard to reason about in the UI.

The rebuild replaces that with a calmer model:

1. Authored content lives in MDX.
2. Dynamic data flows through server routes.
3. Typed schemas define what the site can publish.

That trade turns the portfolio from a demo into a maintainable platform.
