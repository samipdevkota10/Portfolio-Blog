Most portfolio chatbots are prompt wrappers. They sound polished, but they do not earn trust because they cannot show where an answer came from.

This rebuild takes a different path:

- Authoritative content lives in versioned MDX files.
- The assistant retrieves from posts, case studies, and a canonical resume source.
- Responses include citations instead of vague confidence.

## Why retrieval matters

A portfolio site is a narrow domain. That makes it a perfect place to be strict about grounding. If a model cannot answer from the documented corpus, it should say so plainly instead of improvising.

## The engineering shape

The assistant pipeline is intentionally small:

```ts
const documents = await retrieveRelevantChunks(question);
const answer = await generateGroundedAnswer(question, documents);
```

That compact architecture makes evaluation tractable, and it keeps future tool expansion honest.
